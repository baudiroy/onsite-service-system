## Exported API

```js
CUSTOMER_ACCESS_AUDIT_REPOSITORY_RECORD_KEYS
buildCustomerAccessAuditRepositoryRecord(input)
normalizeCustomerAccessAuditRepositoryResult(input)
```

Dependency direction:

- The contract uses the accepted audit event builder to sanitize audit-event-like input.
- The contract uses the accepted writer result normalizer for future repository result normalization.
- It imports no DB, env, app/server, route, controller, provider, AI/RAG, billing, migration, SQL runner, or repository implementation modules.

## Accepted Repository Record Keys

`CUSTOMER_ACCESS_AUDIT_REPOSITORY_RECORD_KEYS` is:

- `eventType`
- `occurredAt`
- `requestId`
- `actorType`
- `organizationId`
- `customerId`
- `caseId`
- `reportId`
- `decision`
- `reasonCode`
- `route`
- `method`
- `source`
- `metadata`

No `id`, `createdAt`, raw DB column, SQL text, query params, or raw JSON payload is generated by this contract.

## Invalid Input Behavior

`buildCustomerAccessAuditRepositoryRecord(input)` accepts only plain audit-event-like objects that can be sanitized by the accepted builder.

Invalid input returns:

```js
{
  ok: false,
  status: 'failed',
  auditWritten: false,
  persisted: false,
  reasonCode: 'audit_event_invalid',
}
```

Invalid input includes:

- `null`
- `undefined`
- scalar values
- arrays
- `Date`
- `Error`
- Buffer-like values
- Promise-like / thenable values
- functions
- class instances
- non-plain objects
- unknown event types

Unknown raw fields are not copied into records. Raw request/header/token/SQL/provider/debug/private sentinels do not leak.

## Metadata Behavior

- Metadata is rebuilt as a new object.
- Only metadata accepted by the audit event builder is kept.
- Raw nested metadata keys such as SQL/debug/private fields are omitted by the builder contract.
- Returned metadata does not share mutable references with caller input.

## Repository Result Normalization

`normalizeCustomerAccessAuditRepositoryResult(input)` delegates to the accepted writer result normalizer.

Accepted normalized shapes:

```js
{ ok: true, status: 'recorded', auditWritten: true, persisted: true }
{ ok: true, status: 'skipped', auditWritten: false, persisted: false, reasonCode }
{ ok: false, status: 'failed', auditWritten: false, persisted: false, reasonCode }
```

Malformed result input normalizes to safe failed result with `reasonCode: 'invalid_writer_result'`.

Raw DB errors, SQL, rows, driver metadata, and parameters are not exposed.

## Immutability And Determinism

- Caller input is not mutated.
- Same input returns deep-equal output.
- Returned objects are newly built.
- Returned metadata does not share mutable references with input metadata.
- No `Date.now`, `new Date`, `Math.random`, crypto randomness, `process.env`, or global state is used.

## Static Boundary

Static guard confirms:

- No IO, listener, fetch/http, SQL execution, mutation query, file write, or process env usage.
- Contract module is not imported by Customer Access routes, controllers, middleware, projection handler/service, app, server, or public routes.
