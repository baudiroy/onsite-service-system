## Final Status Matrix

Recorded normalizes only to:

```json
{
  "ok": true,
  "status": "recorded",
  "auditWritten": true,
  "persisted": true
}
```

Skipped normalizes only to:

```json
{
  "ok": true,
  "status": "skipped",
  "auditWritten": false,
  "persisted": false,
  "reasonCode": "audit_skipped"
}
```

Failed normalizes only to:

```json
{
  "ok": false,
  "status": "failed",
  "auditWritten": false,
  "persisted": false,
  "reasonCode": "invalid_writer_result"
}
```

Normalized booleans are generated by the matrix, not by raw input truthiness.

## Per-Status ReasonCode Behavior

Recorded:

- omits `reasonCode` even if supplied

Skipped accepts only:

- `audit_skipped`
- `audit_not_configured`
- `audit_writer_unavailable`

Skipped fallback:

- incompatible, unknown, raw, SQL-like, token-like, or header-like reasonCodes normalize to `audit_skipped`

Failed accepts only:

- `audit_event_invalid`
- `audit_persistence_failed`
- `audit_writer_unavailable`
- `invalid_writer_result`

Failed fallback:

- incompatible, unknown, raw, SQL-like, token-like, or header-like reasonCodes normalize to `invalid_writer_result`

## Contradictory Input Behavior

Contradictory status/flag combinations normalize safely:

- recorded with valid recorded flags and any reasonCode normalizes to recorded with no reasonCode
- recorded with contradictory `ok`, `auditWritten`, or `persisted` normalizes to failed `invalid_writer_result`
- skipped with contradictory `ok`, `auditWritten`, or `persisted` normalizes to failed `invalid_writer_result`
- failed with contradictory `ok`, `auditWritten`, or `persisted` normalizes to failed `invalid_writer_result`

The normalizer does not preserve raw contradictory flags.

## Boolean Strictness

The normalizer does not treat string or numeric booleans as authoritative flags:

- `"true"`
- `"false"`
- `"1"`
- `"0"`
- `1`
- `0`
- `"yes"`
- `"no"`

Those values do not drive normalized booleans and may produce failed `invalid_writer_result` when they make the input matrix invalid.

## Non-Leakage and Regression

Normalized output includes only:

- `ok`
- `status`
- `auditWritten`
- `persisted`
- `reasonCode` when applicable

Unknown input fields and sensitive raw data from Task2106 do not appear in normalized output.

Preserved Task2106 behavior:

- malformed input does not throw
- caller input remains unchanged
- same input returns deep-equal output across calls
- output is newly built and does not share mutable nested references with input
- no DB/env/app/server/routes/provider/AI/billing/repository/controller/runtime imports
- no IO/listen/fetch/process.env/Date.now/Math.random/crypto randomness
- no runtime integration
