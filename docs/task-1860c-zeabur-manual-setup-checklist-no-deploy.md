## PostgreSQL And Migration Safety

- Use a Zeabur PostgreSQL service or an explicitly approved external PostgreSQL service.
- `DATABASE_URL` must not point to `localhost`, `127.0.0.1`, or any local-only host from inside Zeabur.
- Migrations are manual and must not run inside `npm start`.
- `npm run db:migrate` requires explicit separate approval for the exact target database.
- `npm run db:seed` requires explicit separate approval for the exact target database.
- Do not run migrations against shared, staging, or production databases from generic instructions such as "continue" or "do the next task".
- Migration must run before seed when both are approved.
- Applied migrations must not be edited; shared environments must apply new migrations incrementally.
- Respect paused/inert warnings in `migrations/README.md`, especially for migrations that exist as SQL artifacts but are not approved for runtime use or broad shared-environment apply.
- Do not store or print connection strings, passwords, tokens, provider keys, or secret values in docs, chat, commits, logs, or screenshots.
